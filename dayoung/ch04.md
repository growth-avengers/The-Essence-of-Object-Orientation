# ch04. 역할, 책입, 협력
- 협력
  - 요청과 응답을 통해 협력 관계를 맺음
- 책임
  - 하는 것
  - 아는 것
- 역할
  - 역할을 사용하면 협력 내에서 다른 객체로 대체할 수 있음을 나타내는 표식
  - 역할은 단순성, 유연성, 재사용성을 뒷받침하는 핵심 개념
- 객체의 모양을 결정하는 협력
  - 책임 주도 설계 : 협력에 필요한 책임을 식별하고 객체에 책임을 할당
  - 디자인 패턴
  - 테스트 주도 개발 : 테스트를 먼저 작성하고 테스트를 통과하는 코드를 추가하면서 완성해나가는 방식
